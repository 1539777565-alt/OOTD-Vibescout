# 产品需求文档 (PRD)：Aura 智能穿搭配方引擎

**文档状态：** MVP 优化版交付总结

**一句话定位：** OOTD VibeScout 是一个 AI 穿搭搜索翻译器：上传穿搭图后，将“看得懂但说不出”的审美翻译为小红书中文搜索词、Instagram 英文风格标签与可复用单品配方。

---

## 1. 业务洞察与价值主张 (Why we build this?)

### 1.1 核心痛点挖掘

在小红书、Instagram 等穿搭社区，年轻用户每天被大量高质量穿搭「种草」，但在转化为自身穿搭时面临四类阻碍：

| 痛点 | 说明 |
|------|------|
| **审美认知壁垒（不会拆）** | 用户能感知到「老钱风」或「智性清冷」的氛围（Vibe），但缺乏时尚专业知识，无法将其反向拆解为具体的材质、剪裁和单品组合，导致无法搜索。 |
| **平台语义断层（不会翻译）** | 同一种穿搭氛围在小红书常被表达成「松弛感、山系、通勤感」，在 Instagram 则更可能对应 `gorpcore`、`clean fit`、`quiet luxury` 等英文风格标签。用户不知道跨平台该怎么搜。 |
| **求异心理（不想照搬）** | 用户往往不是要一比一复刻，而是希望保留氛围、替换单品、降低撞款概率。传统以图搜图容易把需求误解为“找同款”。 |
| **长句检索失真（搜不准）** | 如果把「日系 City Boy 皮草帽穿搭奶油色阔腿裤氛围感」整句塞进搜索框，平台容易返回泛化或跑偏内容。用户更需要被拆成“氛围入口”和“单品入口”。 |
| **检索摩擦力高（搜不到）** | 用户需要手动拆图、想词、翻译、换平台搜索，链路冗长且每一步都容易断裂。 |

### 1.2 破局策略 (Value Proposition)

切断传统的 SKU 视觉比对链路。利用多模态大模型（VLM）作为审美意图中枢，将非标的「穿搭图片」降维翻译为高度结构化的「风格 DNA、单品配方、中英双语搜索词」，再拆成多个短搜索入口，实现「视觉输入 → 语义降维 → 风格/单品拆解 → 跨平台稳定搜索入口」的闭环体验。

## 2. 核心系统架构与工作流 (System Workflow)

本产品采用 **Agentic Workflow（智能体工作流）** 理念设计，分为表现层、认知层与执行层。

### 2.1 表现层：极简 Vibe 交互 (Gradio UI)

- **输入侧：** 极简拖拽/上传交互，只保留上传图片与“一键生成跨平台搜索方案”，减少无效选项干扰。
- **输出侧：** 展示三段结构化结果：风格 DNA、可搜索穿搭配方、拆解式搜索入口。采用分组信息行展示小红书与 Instagram 入口，降低结果阅读负担。

### 2.2 认知层：意图翻译中枢 (MiniMax VLM)

接收用户上传图片后，调用 MiniMax 多模态大模型进行深度解析。

**核心杠杆：系统级 Prompt 设计** —— 强制模型剥离表面颜色，专注物理材质与结构，输出限定格式的 JSON：

| 字段 | 含义与示例 |
|------|------------|
| `vibe_keywords` | 提炼 3–4 个核心情绪词（如：City Boy、山系少年感）。 |
| `material_and_cut` | 面料与剪裁诊断（如：硬挺牛津纺、宽松阔腿廓形）。 |
| `core_formula` | **产品灵魂：** 单品配方拆解矩阵，包含品类、中文关键词、英文关键词。 |
| `xhs_query` | 面向小红书中文语境的生活化搜索词。 |
| `instagram_query` | 面向 Instagram 语境的英文搜索表达。 |
| `instagram_hashtags` | 可直接打开的英文风格标签入口。 |

### 2.3 执行层：拆解式稳定搜索入口 (Decomposed Search Cards)

1. 拿到 `Core_Formula` 后，系统将配方转化为中文生活化搜索词和英文风格标签。
2. 不抓取平台内容，不伪造笔记链接，而是生成可解释的「搜索策略入口」。
3. 搜索入口分为两组：
   - **小红书：** 先给「通过氛围找穿搭」入口，再给外套、下装、鞋履、配饰等单品入口。
   - **Instagram：** 先给整体 vibe hashtag，再给单品 hashtag，用于查廓形、配色和造型方式。
4. 通过短搜索词替代长句搜索，避免平台检索结果跑偏。

### 2.4 性能与体验优化

- **MCP 路线复用：** 沿用 OpenClaw / MiniMax MCP 的 `understand_image` 能力，保持真实 VLM 解析。
- **协议握手修复：** 在 MCP initialize 后补充 `notifications/initialized`，保证 tool call 正常执行。
- **同图缓存：** 对上传图片计算 hash，同一张图在当前服务进程中只真实解析一次，重复测试与调试排版时直接返回缓存。
- **Prompt 减负：** 限制输出 3 个关键单品模块，减少模型生成长度和等待时间。

## 3. 产品边界与异常兜底策略 (Fallback Mechanism)

*（AIPM 专项设计，保障 MVP 高可用性）*

面对国内社区严格的反爬与风控，执行层设计强兜底，确保前端**永远不崩溃**：

| 场景 | 策略 |
|------|------|
| **模型超时 / 解析失败** | 拦截 VLM 错误并给出明确提示，不返回固定假结果，避免误导演示效果。 |
| **平台反爬 / 链接失效** | 产品层主动规避抓取：只生成用户主动点击的搜索入口，不解析 HTML、不模拟登录、不承诺真实笔记 URL，保证 demo 稳定。 |
| **重复测试耗时** | 对同一张图片做 session 内缓存，避免每次点击都重新等待 VLM。 |

## 4. 商业化闭环与演进路线图 (Roadmap)

### 阶段一：MVP 验证（当前阶段）

- **核心动作：** 跑通「图片 → 风格 DNA → 单品配方 → 小红书 / Instagram 拆解式搜索入口」链路。
- **业务指标：** 验证 prompt 解析成功率、搜索词可用性、同图缓存命中体验；验证自然语言编程（Vibe Coding）独立交付全栈应用的可行性。

### 阶段二：场景深挖与多模态约束（v1.2 规划）

- **Image + Text 双重输入：** 传图同时支持自然语言约束（如「预算 500 以内」「适合见长辈」「夏季版」），拆解配方时融合更细粒度物理限制。
- **我的衣橱 (Wardrobe AI)：** 第二张图输入用户自有衣物，结合参考图给出基于现有衣物的搭配解法，降低购买焦虑。

### 阶段三：CPS 商业化变现（v2.0 规划）

- 可扩展到淘宝联盟 / 京东联盟 API，将「单品配方」进一步承接到可购买 SKU。
- 保留当前跨平台审美翻译能力作为上游意图层，避免直接退化为普通电商以图搜图。
- **盈利模式：** 依托审美意图推荐促成交易，赚取电商平台 CPS（Cost Per Sale）推广分佣，实现流量变现。
